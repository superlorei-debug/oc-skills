# 开发工作流 (DEV_WORKFLOW.md)

## 日常开发流程

### 1. 任务开始

```
1. 理解需求
   └── 阅读用户请求
   
2. 确认范围
   └── 评估改动大小
   
3. 检查规则
   └── 查阅 ARCHITECTURE.md, AI_RULES.md
```

### 2. 开发执行

```
1. 小步迭代
   └── 分批开发，每次只改必要部分
   
2. 本地测试
   └── 运行脚本验证
   
3. 检查输出
   └── 确认数据正确
```

### 3. 完成交付

```
1. 推送 GitHub
   └── git add . && git commit && git push
   
2. 汇报状态
   └── 告知完成内容
   
3. 观察运行
   └── 确认无异常
```

## 任务类型处理

### 小优化 (< 30分钟)

1. 直接执行
2. 测试验证
3. 推送 GitHub
4. 简单汇报

### 功能开发 (30分钟 - 2小时)

1. 方案设计
2. 分步开发
3. 测试验证
4. 推送 GitHub
5. 详细汇报

### 大改动 (> 2小时)

1. 方案确认
2. 分阶段开发
3. 每个阶段测试
4. 完成后推送
5. 用户验收

## 问题处理

### 遇到错误

```
1. 记录错误
   └── 保存错误信息
   
2. 分析原因
   └── 查看日志/代码
   
3. 尝试修复
   └── 本地测试
   
4. 无法解决
   └── 通知用户，说明情况
```

### 需要用户确认

```
1. 说明情况
   └── 发生了什么
   
2. 分析影响
   └── 有什么后果
   
3. 提供选项
   └── A/B方案
   
4. 等待确认
   └── 不要擅自行动
```

## 交付标准

### 代码交付

- [ ] 本地测试通过
- [ ] 功能正常
- [ ] GitHub 已推送

### 状态汇报

- 完成了什么
- 修改了哪些文件
- 当前系统状态
- 后续建议
