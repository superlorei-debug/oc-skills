# 开发规则 (AI_RULES.md)

## 概述

本文档定义了 AI 开发过程中的核心规则，确保系统稳定性和代码质量。

## 核心原则

### 1. 安全优先

- 不运行未经测试的命令
- 修改前先备份
- 重大操作前确认用户
- 不删除运行数据

### 2. 变更管理

- 每次修改后推送到 GitHub
- 记录变更日志
- 遇到问题及时通知用户

### 3. 测试验证

- 新功能先本地测试
- 确认功能正常后再推送
- 观察运行状态

## 开发流程

### 标准流程

1. **理解需求** - 确认任务目标
2. **阅读规则** - 查阅 ARCHITECTURE.md, AI_RULES.md
3. **小步迭代** - 分批开发，避免大改
4. **测试验证** - 本地运行确认
5. **推送 GitHub** - 及时同步
6. **观察状态** - 确认运行正常

### 禁止事项

- ❌ 未经测试直接改生产环境
- ❌ 删除正在运行的数据
- ❌ 硬编码密钥/密码
- ❌ 大幅重构核心逻辑
- ❌ 同时开发多个功能

## 代码规范

### Python

- 使用 4 空格缩进
- 变量命名: 小写 + 下划线
- 函数命名: 小写 + 下划线
- 常量: 全大写 + 下划线

### JSON

- 字段命名: 驼峰或下划线一致
- 数字类型: 不要用 string (除非要格式化)
- 枚举值: 使用标准词汇 (critical/warning/ok)

### Git

- 提交信息: 简洁描述变更
- 推送频率: 每次重要修改后
- 分支: 保持 master

## 异常处理

### 程序错误

1. 记录错误信息
2. 分析原因
3. 尝试修复
4. 无法解决 → 通知用户

### 数据问题

1. 不删除原始数据
2. 创建备份
3. 修复后验证
4. 记录到 bug_log.md

## 沟通规范

### 常规汇报

- 完成后总结修改内容
- 推送后告知 GitHub 状态
- 异常时主动通知

### 问题升级

遇到以下情况立即通知用户:
- 程序崩溃
- 数据丢失风险
- 安全漏洞
- 系统不可用
