# AI Quant Brain 分支协作规则

> **版本**: v1.0  
> **创建日期**: 2026-03-15  
> **目的**: 保护稳定基座，明确协作边界

---

## 一、分支架构总览

```
master (稳定基座)
    ↑
    ├── feature/* (常规新功能)
    ├── hotfix/* (单点故障修复)
    ├── exp/evolution-v1 (自我进化实验)
    └── release/prod-prep (上线准备)
```

---

## 二、分支定义

### 1. master - 稳定基座分支

| 项目 | 内容 |
|------|------|
| **定位** | 唯一稳定基座，后续正式部署来源 |
| **状态** | 经过观察验证的稳定版本 |
| **允许改动** | 单点故障修复、必要生产修复、文档更新 |
| **禁止改动** | 实验性功能、新功能开发、大范围重构 |
| **合并来源** | hotfix/*, release/prod-prep (验证后) |
| **保护规则** | 必须经过验证才能合并 |

### 2. feature/* - 常规新功能分支

| 项目 | 内容 |
|------|------|
| **用途** | 常规新功能开发 |
| **示例** | feature/macro-page, feature/news-enhancement |
| **允许改动** | 新功能实现、页面增强、模块扩展 |
| **禁止改动** | 生产交易逻辑、核心数据结构 |
| **合并规则** | 完成后必须验证 → PR 到 master |

### 3. hotfix/* - 故障修复分支

| 项目 | 内容 |
|------|------|
| **用途** | 生产/稳定观察期单点修复 |
| **示例** | hotfix/scheduler-python-path, hotfix/freshness-threshold |
| **允许改动** | 最小必要修复、单点故障修复 |
| **禁止改动** | 新功能、实验性代码 |
| **合并规则** | 快速验证 → 合并回 master |

### 4. exp/evolution-v1 - 自我进化实验分支

| 项目 | 内容 |
|------|------|
| **用途** | 自我进化、回测、评分逻辑实验、参数实验 |
| **状态** | 独立于 master 的实验线 |
| **允许改动** | scripts/evolution/, data/evolution/ 实验 |
| **禁止改动** | scripts/grid_bot.py, 生产参数, 主交易逻辑 |
| **合并规则** | 默认不合并 → 仅当实验结果被明确审查通过后 PR 到 master |
| **核心原则** | 不允许自动修改生产参数或生产代码 |

### 5. release/prod-prep - 上线准备分支

| 项目 | 内容 |
|------|------|
| **用途** | 服务器部署准备 |
| **允许改动** | 环境变量、启动脚本、反向代理配置、域名配置、正式看板部署 |
| **禁止改动** | 新功能开发、实验性代码 |
| **合并规则** | 验证完成后合并到 master |

---

## 三、合并规则

### 合并流程

```
feature/* / hotfix/* → 验证 → PR → master
exp/evolution-v1 → 审查通过 → PR → master (仅特殊情况)
release/prod-prep → 验证 → PR → master
```

### 合并原则

1. **必须验证**: 所有合并前必须本地测试通过
2. **最小改动**: hotfix/* 必须最小化改动范围
3. **审查机制**: exp/* 实验结果必须经过审查才能合回 master
4. **禁止自动合并**: 不允许 CI/CD 自动合并到 master

---

## 四、禁止事项

| 分支 | 禁止事项 |
|------|----------|
| master | 实验性功能、新功能、大重构 |
| feature/* | 生产交易逻辑修改 |
| hotfix/* | 新功能、实验代码 |
| exp/evolution-v1 | 自动修改生产参数、生产代码 |
| release/prod-prep | 新功能开发 |


---

## 七、协作默认规则

| 场景 | 分支 | 说明 |
|------|------|------|
| 稳定基座 | master | 唯一稳定来源 |
| 新功能开发 | feature/* | 必须验证后才能合回 |
| 故障修复 | hotfix/* | 最小改动，快速验证 |
| 自我进化实验 | exp/evolution-v1 | 默认不合并 |
| 上线准备 | release/prod-prep | 部署环境准备 |

---

## 八、注意事项

1. **不要直接在 master 上开发**: 所有新功能必须从 feature/* 开始
2. **实验线隔离**: exp/evolution-v1 必须与 master 保持独立
3. **热修复优先**: 发现问题优先用 hotfix/* 处理
4. **验证优先**: 任何合并前必须验证通过
